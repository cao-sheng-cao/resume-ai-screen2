简历岗位匹配评分系统 v1.0.33 密钥保存错误热修复报告

一、问题现象
用户在保存接口密钥时出现错误：ReferenceError: setStoredApiKey is not defined。

二、原因
v1.0.29 之后为了降低 vibe slop 风险，把 API Key 存储逻辑改为 safeStorage 加密保存；v1.0.30 模块化拆分后，main.js 中仍调用 setStoredApiKey / getStoredApiKey / removeStoredApiKey，但相关函数没有作为模块正确引入，导致运行时报 ReferenceError。

三、本次修复
1. 新增 src/main/services/secure-settings.js。
2. 将 canUseSafeStorage、getStoredApiKey、setStoredApiKey、removeStoredApiKey 放入 secure-settings 模块。
3. main.js 显式引入 secure-settings 模块。
4. scripts/selfcheck.js 新增密钥函数导入检查，避免类似问题再次漏检。
5. 版本号升级为 v1.0.33。
