简历岗位匹配评分系统 v1.0.29 反 Vibe Slop 工程整理升级报告

一、本次升级结论
本次升级完成。静态自检全部通过，失败项为 0。

二、本次实际完成的工程整理
1. JSON 本地存储改为原子写入：先写临时文件，再 rename 覆盖目标文件，降低异常退出导致 JSON 损坏的概率。
2. API Key 存储升级：优先使用 Electron safeStorage 加密保存；如果当前系统不支持 safeStorage，才回退明文保存。
3. 旧版明文 API Key 自动迁移：读取旧 settings.json 时，如果发现旧版明文密钥且 safeStorage 可用，会自动迁移为加密密钥。
4. 备份安全处理：默认备份不会导出 API Key，也不会导出 safeStorage 密文；只有用户明确勾选导出密钥时，才导出可跨电脑恢复的明文密钥，并继续依赖前端二次确认。
5. 候选人卡片与排行榜数据结构分离：项目内新增 candidates 数据，候选人卡片不再直接依赖排行榜数组；排行榜继续作为排序/筛选视图。
6. 增加 npm run selfcheck：新增 scripts/selfcheck.js，可在本地或 CI 中执行静态自检。
7. 增加 npm run check：同时检查 JS 语法与项目静态一致性。
8. 继续保留历史功能：OCR双通道、严格度隔离排行榜、ChatGPT式候选人卡片、设置折叠、Ctrl缩放、结果关键信息突出显示。

三、没有一次性大拆模块的原因
renderer.js 和 main.js 仍然较大。本次没有把它们彻底拆成多个运行模块，原因是该应用已经积累了较多功能，直接大拆会显著增加回归风险。本版先完成更安全的工程整理：数据安全、密钥安全、自检脚本、候选人/排行榜数据语义分离。后续可以在 v1.0.30 单独做模块化拆分。

四、建议下一版继续做的模块化拆分
1. renderer.js 拆分为 settings、projects、candidateCards、leaderboard、analysisResult、resumeUpload、backup、uiState、utils。
2. main.js 拆分为 storage、secureSettings、aiClient、fileParser、ocrService、backupService、windowControl。
3. 将 selfcheck.js 加入 GitHub Actions 构建流程，在打包前先运行 npm run check。
4. 候选人数量增长后，将 projects.json 升级为 SQLite。
